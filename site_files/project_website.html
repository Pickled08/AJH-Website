This Website!

You may have guessed that this is a project im working on at the moment but here is the project page for it!

Overview

This website is built using the Flask framework for Python. It uses a combination of HTML, CSS and Jinja templating to create the pages you see. The website is hosted on a server using Flask and Nginx on a Pi 2B with a cloudflare link to get it working with a domain. All apps on the pi are using docker to make it easier to manage as i have portainer and other custom scripts to auto update and reboot programe

The website features a blog, project pages, an about page and a contact form. It also has user authentication for logging in and managing accounts.

All source code is available on my GitHub apart from the nginx configuration as Im not sure how safe it is to share.

Flask

I chose to use flask mainly because i have used it before and it's a great framework for building web applications in Python. It's simple to use and has a lot of features that make it easy to build complex applications. The benifit of using python for this is due to the many libraries available for web development and data processing at my fingertips that are easy to get running (I am also best at python so that helps ;)). I have made good use of flask libraries such as Flask-WTF for form handling and Flask-Login for user authentication.

Nginx

Im using Nginx to acts as a reverse proxy for the Flask application, handling incoming requests and routing them to the appropriate backend services. This makes it so much easier to manage the server and handle traffic efficiently as im not directly accesing the flask backend. This is why I can have the maintanance page working whilest the flask server is down and it also handels the cloudflare Certificates for me. As I said makes it some much better than just using flask, and if I ever need to change the server or add more services, it's much easier to do so with Nginx.

Docker

Docker is being used to containerize the Flask application, making it easier to deploy and manage. Each component of the website (Flask app, Nginx server) runs in its own container, ensuring that they are isolated from each other and can be managed independently. This setup allows for easy updates and scalability, as I can simply update the Docker images or add more containers as needed without affecting the overall system. This also means I can switch between different host machines easily and i know it just works
